# 👶 Baby Diary - Многопользовательское приложение для дневника ребенка

![Python](https://img.shields.io/badge/Python-3.11+-blue.svg)
![Docker](https://img.shields.io/badge/Docker-20.10+-blue.svg)
![License](https://img.shields.io/badge/License-MIT-green.svg)

Полнофункциональное веб-приложение для отслеживания событий из жизни ребенка: сон, пробуждение, кормление и покупки. Поддерживает множество пользователей с уникальными ID и полной синхронизацией данных.

## 🎯 Основные возможности

- **👥 Мультипользовательская система** - каждый пользователь имеет уникальный UUID
- **📝 Отслеживание событий**:
  - 😴 Сон (с определением ночного сна)
  - 🌅 Пробуждение
  - 🍼 Кормление
  - 🛍️ Покупки
- **📄 Система заметок** - отдельный раздел для текстовых заметок с временем создания
- **📊 Аналитика**:
  - 7-дневные графики сна и бодрствования (Chart.js)
  - Статистика по дням и неделям
  - Детальная история событий
- **💾 Сохранение данных**:
  - Сервер хранит данные в JSON
  - Синхронизация между устройствами
  - Экспорт/импорт данных
- **⚙️ Управление аккаунтом**:
  - Регистрация по UUID
  - Копирование ID в буфер обмена
  - Удаление аккаунта со всеми данными

## 🚀 Быстрый старт

### Способ 1: Локально (без Docker)

```bash
# Клонируйте репозиторий
git clone https://github.com/YOUR_USERNAME/baby-diary.git
cd baby-diary

# Запустите сервер
python server.py

# Откройте в браузере
http://localhost:8000
```

### Способ 2: Docker Compose

```bash
# Клонируйте репозиторий
git clone https://github.com/YOUR_USERNAME/baby-diary.git
cd baby-diary

# Запустите приложение
docker-compose up -d

# Откройте в браузере
http://localhost:8000
```

### Способ 3: На TrueNAS

```bash
# Скопируйте на TrueNAS
scp -r /path/to/baby-diary root@YOUR_TRUENAS_IP:/mnt/tank/apps/

# Подключитесь по SSH
ssh root@YOUR_TRUENAS_IP

# Запустите
cd /mnt/tank/apps/baby-diary
docker-compose up -d
```

См. [DOCKER-DEPLOYMENT.md](DOCKER-DEPLOYMENT.md) для подробных инструкций.

## 🔧 Технический стек

### Frontend
- **HTML5** - семантическая разметка
- **CSS3** - гибкий responsive дизайн
- **Vanilla JavaScript** - без фреймворков и зависимостей
- **Chart.js** - интерактивные графики (подключается через CDN)

### Backend
- **Python 3.11** - язык программирования

- **JSON** - формат сохранения данных
- **UUID v4** - уникальные идентификаторы пользователей

## 🔐 Безопасность

- **Изоляция данных** - каждый пользователь видит только свои данные
- **UUID v4** - криптографически стойкий идентификатор (не последовательный)
- **Нет аутентификации** - использует UUID как простой механизм идентификации
- **Локальное хранилище** - данные хранятся на вашем сервере

⚠️ **Примечание**: Для production используйте HTTPS и добавьте авторизацию по паролю!

## 📝 Использование

### Запись события
1. Откройте приложение
2. Зарегистрируйтесь или войдите с существующим ID
3. На вкладке "Запись" нажмите кнопку события (😴 / 🌅 / 🍼 / 🛍️)
4. Опционально добавьте заметку и подтвердите

### Просмотр истории
- Вкладка "История" показывает все события с вычисленной продолжительностью
- Ночной сон (23:00-07:00) помечается как 🌙 Ночной сон

### Аналитика
- Вкладка "Анализ" показывает графики за 7 дней
- Автоматически вычисляются средние значения

### Заметки
- Вкладка "Заметки" для быстрых заметок
- Все заметки синхронизируются с сервером

### Экспорт данных
- Вкладка "Настройки" → кнопка 📥 Загрузить JSON
- Можно импортировать данные обратно

## 🐛 Известные ограничения

- Только одна сессия на браузер (разные браузеры = разные сессии)
- Нет шифрования данных (для production добавить)

## 🆘 Решение проблем

### Приложение не открывается
- Проверьте что сервер запущен (Terminal → `python server.py`)
- Откройте http://localhost:8000 в браузере
- Проверьте консоль браузера на ошибки (F12 → Console)

### Данные не сохраняются
1. Откройте DevTools (F12)
2. Перейдите на Network
3. При нажатии кнопки события проверьте что запрос к `/save` успешен (200 OK)
4. Если ошибка 404 - сервер не запущен

### UUID не копируется
- Это работает только на HTTPS или localhost
- На HTTP копирование может быть заблокировано браузером
